⚡ Automação

Workflow Automatizado

O projeto agora tem scripts e tasks configurados para automatizar build, deploy e sync Android — tudo com confirmações de segurança antes de executar.

🗂️ Arquivos criados

ArquivoO que faz
.vscode/tasks.jsonTasks de build/deploy acessíveis com Ctrl+Shift+B
tools/deploy.batScript Windows com menu interativo (clique duplo)
tools/deploy.ps1Script PowerShell colorido e robusto
.env.exampleTemplate seguro para tokens e variáveis

🔄 Fluxo real do projeto

[VS Code Agent / Blackbox] → gera e revisa código ↓ [Ctrl+Shift+B → Build] → npm run build (TypeScript + Vite) ↓ [Git Commit + Push] → git add -A && git commit && git push ↓ [Deploy Vercel] → npx vercel --prod ↓ [Sync Android] → npx cap sync android → Android Studio → APK
💡 Regra de ouro: Use o agente do VS Code (Blackbox) para escrever e revisar código. Use os scripts/tasks para executar e fazer deploy. Isso equilibra o consumo dos créditos.

📜 Scripts Windows

Scripts de Deploy para Windows

Dois scripts prontos para usar no Windows — com menu interativo e confirmações antes de cada ação crítica.

🦇 deploy.bat — Script Batch (mais simples)

Clique duplo no arquivo ou execute no terminal:

tools\deploy.bat

Menu com 5 opções:

  • [1] Deploy Web (Build + Vercel)
  • [2] Sync Android (Build + cap sync)
  • [3] Deploy Completo (Build + Git + Vercel)
  • [4] Apenas Build
  • [5] Apenas Git Push

💙 deploy.ps1 — Script PowerShell (mais robusto)

Execute no terminal do VS Code:

.\tools\deploy.ps1

Ou clique direito no arquivo → "Executar com PowerShell"

Vantagens sobre o .bat:

  • Saída colorida (verde = sucesso, vermelho = erro)
  • Tratamento de erros mais robusto
  • Opção extra: Build do sub-app Arquitetura
💡 Primeira vez com PowerShell? Se aparecer erro de "execução de scripts desabilitada", execute no terminal como Administrador:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

⌨️ Executar direto no terminal do VS Code

# Abrir terminal: Ctrl + ` (acento grave) # Script Batch: tools\deploy.bat # Script PowerShell: .\tools\deploy.ps1 # Ou comandos diretos: npm run build npx vercel --prod npx cap sync android
🔐 Variáveis de Ambiente

Configurando Variáveis de Ambiente

Tokens e senhas nunca devem ficar no código. Use o arquivo .env para guardá-los com segurança.

1

Copie o arquivo de exemplo

No terminal do VS Code: copy .env.example .env

2

Edite o .env com seus tokens reais

Abra o .env e substitua os valores de exemplo pelos seus tokens reais (Vercel, PushinPay, Blackbox, etc.)

3

O .env está protegido pelo .gitignore

O arquivo .env nunca será enviado ao GitHub. Apenas o .env.example (sem valores reais) é versionado.

🔑 PushinPay

Configurando o PushinPay (PIX)

O PushinPay é o gateway de pagamento PIX usado no projeto. Você precisa de uma conta e do token de API.

1

Criar conta no PushinPay

Acesse pushinpay.com.br e crie sua conta. É gratuito para começar.

2

Obter o Token de API

No painel PushinPay: Configurações → API → Gerar Token. Copie o token gerado.

3

Adicionar o token no Vercel

Veja a seção ⚙️ Variáveis no Vercel abaixo para saber como adicionar.

4

Configurar o Webhook no PushinPay

No painel PushinPay, configure a URL de webhook:
https://lux-driver-assistent-18y8.vercel.app/api/webhook-pushinpay

⚠️ Importante: O webhook é chamado automaticamente pelo PushinPay quando o pagamento é confirmado. Sem o webhook configurado, o código não será enviado ao cliente.

🧪 Testar o webhook manualmente

Para testar se o webhook está funcionando, faça uma requisição GET:

curl https://lux-driver-assistent-18y8.vercel.app/api/webhook-pushinpay # Deve retornar: {"status":"webhook endpoint ativo"}
📱 WhatsApp

Configurando o WhatsApp (Evolution API)

O código de ativação é enviado automaticamente via WhatsApp usando a Evolution API. Você precisa de uma instância configurada.

📋 Variáveis necessárias

VariávelExemploDescrição
EVOLUTION_API_URLhttps://sua-api.evolution.comURL da sua instância Evolution API
EVOLUTION_API_KEYabc123xyzChave de autenticação da API
EVOLUTION_INSTANCElux-driverNome da instância WhatsApp
LUX_SECRETLUX_SAULO_2025_DRIVERSegredo para geração de códigos (padrão já definido)
1

Opção A: Evolution API hospedada (recomendado)

Use um serviço como evolution-api.com ou hospede você mesmo no Railway/VPS. Obtenha a URL, API Key e nome da instância.

2

Opção B: Hospedar localmente (desenvolvimento)

Clone o repositório da Evolution API e rode localmente com Docker. Use ngrok para expor o endpoint.

3

Conectar o WhatsApp à instância

No painel da Evolution API, escaneie o QR Code com o WhatsApp do número que vai enviar os códigos.

4

Adicionar as variáveis no Vercel

Veja a seção ⚙️ Variáveis no Vercel abaixo.

💡 Sem Evolution API? O sistema ainda funciona — o código é gerado e retornado no webhook, mas não é enviado automaticamente. Você pode enviar manualmente pelo Admin Dashboard clicando em 💬.
⚙️ Vercel

Configurando Variáveis de Ambiente no Vercel

As variáveis de ambiente das funções serverless (pagamento, webhook) precisam ser configuradas no painel do Vercel — não no arquivo .env local.

1

Acesse o painel do Vercel

vercel.com/dashboard → clique no projeto lux-driver-assistent

2

Vá em Settings → Environment Variables

Menu lateral: Settings → Environment Variables

3

Adicione cada variável

Clique em "Add New" para cada variável abaixo. Selecione os ambientes: Production + Preview + Development

4

Faça um novo deploy

Após adicionar as variáveis, faça um novo deploy para que entrem em vigor: npx vercel --prod

📋 Todas as variáveis necessárias

VariávelObrigatóriaDescrição

EVOLUTION_API_URLRecomendadoURL da Evolution API para envio de WhatsApp
EVOLUTION_API_KEYRecomendadoChave de autenticação da Evolution API

⚠️ Atenção: Sem o PUSHINPAY_TOKEN, o checkout retorna erro 502. Sem as variáveis da Evolution API, o código é gerado mas não enviado via WhatsApp — você precisará enviar manualmente.

✅ Como verificar se está funcionando

  1. Acesse checkout.html
  2. Selecione o plano Mensal
  3. Digite seu número de WhatsApp
  4. Clique em "Gerar PIX" — deve aparecer o QR Code
  5. Se aparecer erro, verifique o PUSHINPAY_TOKEN no Vercel
📊 Admin Dashboard

Como usar o Dashboard

O dashboard é um arquivo HTML local que salva tudo no localStorage do navegador. Sempre abra no mesmo navegador para não perder os dados.

⚠️ Importante: Os dados ficam salvos no navegador (localStorage). Não limpe o histórico/cache do navegador ou os dados serão perdidos. Use sempre o mesmo navegador (ex: Chrome).

📊 Aba Dashboard

  • Mostra total de assinantes, receita total, MRR (receita mensal recorrente)
  • Gráfico de receita por plano
  • Lista dos últimos assinantes cadastrados
  • Índice do próximo código a ser gerado

👥 Aba Assinantes

  • Adicionar: clique em "+ Novo Assinante" → preencha nome, WhatsApp, plano → o código é gerado automaticamente
  • Buscar: use a barra de pesquisa por nome ou WhatsApp
  • Filtrar: por status (ativo/expirado/trial)
  • Enviar código: clique em 💬 → abre WhatsApp com mensagem pronta
  • Editar/Excluir: ícones ✏️ e 🗑️ em cada linha

🔑 Aba Códigos

  • Gera um intervalo de códigos (ex: índices 1 a 10)
  • Códigos já usados aparecem em vermelho
  • Códigos disponíveis aparecem em verde
  • Botão "Copiar todos" para copiar a lista

💰 Aba Receita

  • Total recebido, MRR, ARR projetado
  • Histórico de pagamentos

📤 Exportar CSV

Botão no topo do dashboard → baixa planilha com todos os assinantes. Abra no Excel ou Google Sheets.

🔐 Sistema de códigos

Como funciona o sistema de códigos

🔑 Formato

Todos os códigos seguem o padrão: LUX-XXXX-XXXX

Exemplo: LUX-A3KM-P7QR

📊 Capacidade

  • 500 códigos únicos disponíveis (índices 1 a 500)
  • Cada código é gerado a partir de um índice sequencial
  • O dashboard controla qual índice usar a seguir

✅ Validação no app

O app valida o código localmente (sem internet). O algoritmo é o mesmo no app e no dashboard — garantindo que apenas códigos gerados por você funcionem.

⚠️ Segurança: Nunca compartilhe o arquivo admin-dashboard.html ou code-generator.html com ninguém. Eles contêm o algoritmo de geração de códigos.
💡 Dicas

Dicas importantes

💾 Backup dos dados

Os dados ficam no localStorage do navegador. Para fazer backup:

  1. Abra o Dashboard
  2. Clique em "Exportar CSV"
  3. Salve o arquivo em local seguro (Google Drive, etc.)
  4. Faça isso semanalmente

📱 Acessar de outro computador

O localStorage é local — não sincroniza entre computadores. Se precisar acessar de outro PC:

  1. Exporte o CSV do computador atual
  2. No novo computador, importe os dados manualmente
  3. Ou use sempre o mesmo computador para o dashboard
